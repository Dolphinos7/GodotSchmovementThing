# GodotSchmovementThing
Idk messing around in game dev again for fun

Structure: FE-style tactical layer (positioning, who-attacks-whom) for planning, resolved each turn by one continuous movement-skill run instead of stat-roll combat.

Stakes split: the run's overall pace/cleanliness drives a shared hit-chance modifier for everyone acting that turn; per-unit "crit coins" are optional harder lines through the course, skippable without penalty (no coin = still a normal hit), present for high-skill expression rather than gating basic competence.

Difficulty curve: coins start easy/forgiving, become genuine skill showcases on higher difficulties — explicit tuning deferred.

Stat tie-in (tentative): enemy stats shape overall course harshness, ally stats shape how forgiving their own coin is — ties FE-style leveling back into the movement layer rather than leaving it cosmetic.

Identity split: characters (not weapons) are what you level up and get attached to; sentient weapons supply movement kits and are framed as what you're piloting during a "hacking" abstraction of the parkour run, decoupling it from needing to match each battlefield's literal geography.
